# Research

## Decisão: protótipo front-end local

**Rationale**: o objetivo é testar visual, fluxo de convite, RSVP e Pix rapidamente. Um backend real seria prematuro sem dados definitivos da família.

**Alternativas consideradas**:
- Persistir RSVP em API própria: descartado para este teste inicial.
- Usar serviço externo de formulário: descartado para manter o protótipo autônomo.

## Decisão: estética inspirada em Casar.com, adaptada para aniversário infantil

**Rationale**: Casar.com transmite evento especial, organização e convite digital elegante. O tema infantil precisa ser mais suave, com formas orgânicas, ilustrações, cards e CTAs acolhedores.

## Decisão: imagens externas decorativas

**Rationale**: usar imagens genéricas/externas de bebês/festa e balões evita depender de fotos pessoais. O design não deve parecer banco de imagem pesado; as imagens são apoio visual.
